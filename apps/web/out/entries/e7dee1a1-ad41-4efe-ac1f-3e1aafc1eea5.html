HotLap: Useful PostgreSQL Analytics Queries

article · hotlap, sql, analytics, debugging, iracing · 3/21/2026
Working SQL queries for analyzing HotLap telemetry data in Neon PostgreSQL. Covers session progression, corner-level metric extraction from JSONB, progressive guide maturity, live log debugging, and track summary stats. Run via: source .env.local && psql "$DATABASE_URL" -c "..."

HotLap: Useful PostgreSQL Analytics Queries

All queries run against Neon PostgreSQL. Pattern: source .env.local && psql "$DATABASE_URL" -c "..."

Schema: sim_racing.sessions, sim_racing.laps, sim_racing.tracks, sim_racing.track_guides, sim_racing.live_log

Corner metrics are stored as JSONB in laps.lap_analysis->'corners' (array). To query them, unnest with jsonb_array_elements(l.lap_analysis->'corners') as corner.

Session Progression by Track

-- Best lap, total sessions, total laps per track
SELECT 
  t.name as track,
  MIN(l.lap_time) as best_lap,
  COUNT(DISTINCT s.id) as sessions,
  COUNT(l.id) as total_laps
FROM sim_racing.sessions s
JOIN sim_racing.tracks t ON s.track_id = t.id
JOIN sim_racing.laps l ON l.session_id = s.id
WHERE l.lap_time > 30
GROUP BY t.name
ORDER BY sessions DESC;

All Sessions with Lap Stats (chronological)

SELECT 
  s.id,
  t.name as track,
  s.created_at::date as date,
  COUNT(l.id) as laps,
  MIN(l.lap_time) as best_lap,
  AVG(l.lap_time) as avg_lap
FROM sim_racing.sessions s
JOIN sim_racing.tracks t ON s.track_id = t.id
JOIN sim_racing.laps l ON l.session_id = s.id
WHERE l.lap_time > 30
GROUP BY s.id, t.name, s.created_at
ORDER BY s.created_at ASC;

Corner Metric Trends Over Sessions (TB/LQ scores)

-- Average trail brake and line quality scores per session at a given track
SELECT 
  s.id as session_id,
  s.created_at::date as date,
  MIN(l.lap_time) FILTER (WHERE l.lap_time > 30) as best_lap,
  COUNT(*) as corner_observations,
  AVG((corner->>'trail_brake_score')::float) as avg_tb,
  AVG((corner->>'line_quality_score')::float) as avg_lq
FROM sim_racing.sessions s
JOIN sim_racing.tracks t ON s.track_id = t.id
JOIN sim_racing.laps l ON l.session_id = s.id
  AND l.lap_analysis IS NOT NULL,
  jsonb_array_elements(l.lap_analysis->'corners') corner
WHERE t.name = 'Summit Point Raceway'
GROUP BY s.id, s.created_at
ORDER BY s.id;

Full Corner Breakdown from a Single Lap

-- Per-corner metrics for the best lap of a session
SELECT 
  l.lap_time,
  corner->>'label' as corner,
  (corner->>'trail_brake_score')::int as tb,
  (corner->>'line_quality_score')::int as lq,
  (corner->>'min_speed_kph')::float as min_speed,
  (corner->>'coasting_time')::float as coast_s,
  corner->'trail_brake_components'->>'brake_taper' as taper,
  corner->'trail_brake_components'->>'connection' as connection,
  corner->'trail_brake_components'->>'rotation_readiness' as rotation,
  corner->'line_quality_components'->>'steering_smoothness' as steer_smooth,
  corner->'line_quality_components'->>'steering_corrections' as steer_corr
FROM sim_racing.laps l,
  jsonb_array_elements(l.lap_analysis->'corners') corner
WHERE l.session_id = 113
  AND l.lap_time = (SELECT MIN(l2.lap_time) FROM sim_racing.laps l2 WHERE l2.session_id = 113 AND l2.lap_time > 30)
ORDER BY (corner->>'brake_point_dist')::float;

Progressive Guide Maturity by Track

SELECT 
  tg.track,
  tg.car,
  tg.session_count,
  tg.best_lap_s,
  tg.first_session_at::date,
  tg.last_session_at::date,
  tg.status,
  jsonb_array_length(tg.guide_json->'corners') as corner_count
FROM sim_racing.track_guides tg
WHERE tg.session_count > 0
ORDER BY tg.session_count DESC;

Live Log: What the Driver Saw In-Session

-- All banners shown during a session (real-time feedback)
SELECT tick, data 
FROM sim_racing.live_log
WHERE session_id = 112 AND event_type = 'display_banner' 
ORDER BY tick;

-- Compare live min_speed vs guide speed per corner
SELECT 
  data->>'label' AS corner, 
  data->>'min_speed' AS live_speed, 
  data->>'guide_min_speed' AS guide
FROM sim_racing.live_log
WHERE session_id = 112 AND event_type = 'corner' 
ORDER BY tick;

-- Full event timeline for a session
SELECT tick, event_type, data 
FROM sim_racing.live_log
WHERE session_id = 112 
ORDER BY tick;

Early vs Recent Score Comparison

-- Compare metric averages between early and recent sessions at a track
-- Adjust the session ID threshold to split early/recent
SELECT 
  CASE WHEN s.id <= 94 THEN 'early' ELSE 'recent' END as phase,
  AVG((corner->>'trail_brake_score')::float) as avg_tb,
  AVG((corner->>'line_quality_score')::float) as avg_lq,
  COUNT(*) as observations
FROM sim_racing.sessions s
JOIN sim_racing.tracks t ON s.track_id = t.id
JOIN sim_racing.laps l ON l.session_id = s.id
  AND l.lap_analysis IS NOT NULL,
  jsonb_array_elements(l.lap_analysis->'corners') corner
WHERE t.name = 'Summit Point Raceway'
GROUP BY 1
ORDER BY 1 DESC;

Notes on Schema

  • laps.lap_analysis is JSONB with keys: corners[], brake, g_force, steering, throttle, proximity, coasting_pct, sector_times, traffic
  • Corner objects: label, trail_brake_score, line_quality_score, trail_brake_components{}, line_quality_components{}, min_speed_kph, gear_at_apex, coasting_time, brake_point_dist, speed_ratio, overlap_duration_s, taper_duration_s
  • track_guides.guide_json has a corners[] array with reference values (ref_min_speed, ref_brake_point_m, ref_gear, confidence, session_count, name from Lovely Track Data)
  • live_log.data is JSONB, structure varies by event_type (corner, lap, display_banner, display_next_corner, sector, incident)
  • Filter lap_time > 30 to exclude aborted laps and outliers
