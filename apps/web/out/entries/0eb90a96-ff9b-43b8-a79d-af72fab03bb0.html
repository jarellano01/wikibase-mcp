Wiki Idea Promotion Model: Aggregate Entries Supersede Source Entries

idea · meta, wiki, knowledge-management · 3/21/2026
A tiered idea promotion model where multiple related entries (a, b, c) can be aggregated into a higher-level synthesis (d). The aggregate links back to its sources; sources are tagged as superseded and filtered from default searches, but preserved for lineage.

Wiki Idea Promotion Model: Aggregate Entries Supersede Source Entries

As a wiki grows, related entries accumulate and create clutter. Rather than deleting them, promote them into aggregates that supersede the originals.

Levels

| Level | Description | Example | |-------|-------------|---------| | 1 — Observation | Raw note from a specific project or context | "BGC uses nuqs for filter state" | | 2 — Pattern | Synthesized from 2+ observations | "URL-synced filter state pattern" | | 3 — Principle | Distilled from multiple patterns, broadly applicable | "Externalize ephemeral UI state to the URL" |

Conventions (using tags + content links)

Source entries (a, b, c) — add tag superseded and note the aggregate in content:

tags: [...original tags, "superseded"]
> Superseded by: [Aggregate Entry Title] {uuid}

Aggregate entry (d) — add tag aggregate and list sources:

tags: [..., "aggregate"]
> Emerged from:
> - [Entry A title] {uuid}
> - [Entry B title] {uuid}
> - [Entry C title] {uuid}

Search behavior intent

  • Default searches should prefer entries without the superseded tag
  • Explicitly searching superseded or browsing by ID still surfaces them
  • This keeps the "active" knowledge surface clean without losing lineage

Why keep superseded entries?

  • Lineage: shows how the idea evolved and what specific cases informed it
  • Context recovery: if the aggregate is too abstract, the originals have the concrete examples
  • Non-destructive: avoids the loss of nuance that comes with deletion

Promotion triggers

Consider aggregating when:

  • 3+ entries share overlapping tags and similar summaries
  • An entry is referenced repeatedly across projects
  • A pattern has been validated across multiple clients/contexts
